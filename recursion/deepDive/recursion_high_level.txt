When are some indicators I should use recursion?

- Top to bottom hierarchy
  -- Need to explore alternatives that all use the same 
     sort of processing

How do I solve a recursion problem?

- Identify base case(s)
- Identify the "normal" case processing,
  which will involve recursive calls to smaller problems

What is the runtime and space complexity of a recursive algorithm?

- Runtime: O(number of nodes in the search tree) = O(branching-factor ^ depth-of-tree)
- Space: O(size of the stack + output_size) = O(depth-of-tree + output_size)

Problem 1

In a simplified game of American football, teams score points by either achieving a 
touchdown (7 points) or a field goal (3 points).

Given the score for a single team, please return how many different permutations of 
touchdowns and/or field goals exist in order to arrive at that score.

Input: Integer
Output: Integer

Examples
Input:  10
Output: 2

Explanation: For a score of 10, the output would be 2. The 2 ways to arrive at
this score is to:

1) Score a field goal (3 points) and then touchdown (7 points)
2) Score a touchdown (7 points) and then field goal (3 points)


Input: 21
Output: 2

Explanation: For a score of 21, the output would be 2. The 2 ways to arrive at
this score is to:

1) Score 7 field goals (7 * 3 points)
2) Score 3 touchdowns (3 * 7 points)

Input: 24
Output: 5

1) 8  field goals (8 * 3)
2) 3 TDs and 1 FG (3*7 + 3)
3) 2 TDs, 1FG, 1 TD (2*7 + 3 + 7)
etc.

Constraint:

0 <= input_score <= 90

  return answer

Problem 2

Given a string str of lowercase alphabetical characters, return the set of all 
variations of those characters in upper AND lowercase.

Input: str (String)
Output: [Str] (Array of Strings)


Example
Input: "abc"
Output: ["ABC", "ABc", "AbC", "aBC", "Abc", "aBc", "abC", "abc"]

Example:
Input: ""
Output: [""]

Constraints
Time Complexity: O(2^N)
Space Complexity: O(2^N)

The order of the strings in the final result does not matter.

In the basic solution, all the input characters will be lowercase letters.
